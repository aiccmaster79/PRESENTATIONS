Account Permissions

How user accounts and permissions protect files

OCN NI Level 2–3 — IT support / service delivery

Learning Objectives

By the end of this session, you will be able to:

  • Explain why separate user accounts improve security
  • Compare standard user vs administrator rights
  • Apply folder permissions (Read, Modify, Full Control)
  • Demonstrate how denied access protects shared files
  • Follow least-privilege practice in support work

Why Permissions Matter

Accounts answer who you are. Permissions answer what you may do. Together they stop accidental deletes and unauthorised access.

Tip

Least privilege: give the minimum access needed for the job.

Standard vs Admin

Standard user

  • Day-to-day work account
  • Installs need elevation
  • Limits malware damage
  • Preferred for staff PCs

Administrator

  • Can change system settings
  • Install software & drivers
  • Manage other accounts
  • Use only when required

Common NTFS Rights

PermissionWhat it allowsTypical use
ReadOpen/view filesPolicies, handbooks
Read & executeRun programsShared apps
ModifyEdit/delete own workTeam project folder
Full controlChange perms tooAdmins / owners
DenyBlocks accessUse sparingly — overrides Allow

Tip

On shared folders, prefer groups over individual names.

Demo Sequence

  1. Create two users: StaffUser (standard) and AdminDemo (admin)
  2. Create a shared folder with a sample file
  3. Set StaffUser to Read only; leave AdminDemo with Modify
  4. Log in as StaffUser — open works, save/delete should fail
  5. Log in as AdminDemo — edit succeeds
  6. Discuss ticket scenarios: wrong access vs broken app

Q1: Least privilege means…

AEveryone gets Full Control
BGive only the access needed for the role
CNever create user accounts
DShare one admin password
Answer: B — Minimum necessary rights reduce mistakes and malware impact.

Q2: A staff user can open a file but cannot save changes. Likely cause?

ABroken monitor
BRead-only permission on the folder/file
CWrong keyboard layout
DDHCP failure only
Answer: B — Read without Modify/Write blocks saves.

Q3: Why prefer groups on a shared folder?

AGroups look prettier
BEasier to grant/revoke access as people join or leave
CNTFS ignores users
DGroups disable auditing
Answer: B — Manage membership once; permissions stay stable.
