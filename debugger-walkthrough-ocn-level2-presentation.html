Debugger Walkthrough

Find program faults with breakpoints and step-through

Learning Objectives

By the end of this session, you will be able to:

  • Explain what a debugger does compared with print-debugging
  • Set a breakpoint and run to it in VS Code or Visual Studio
  • Step over/into code and watch variable values change
  • Locate a simple logic or off-by-one fault using the debugger
  • Describe when to use the Call Stack and Watch panes

Why Use a Debugger?

A debugger lets you pause a running program, inspect variables, and step line-by-line — faster and clearer than scattering print() everywhere.

Tip

Reproduce the bug first, then breakpoint just before the suspicious line — don’t step from line 1 of a huge file.

Debugger Toolkit

Breakpoint

Pause when this line is about to run

Step Over

Run this line; don’t dive into functions

Step Into

Enter a function call to see inside

Continue

Run until the next breakpoint

Watch / Locals

See live variable values

Call Stack

Which functions called you here?

Demo Sequence

  1. Open the prepared Python sample with a known bug (e.g. wrong total or off-by-one loop)
  2. Set a breakpoint on the first suspect line (red dot in the gutter)
  3. Start Debugging (F5) — program pauses at the breakpoint
  4. Inspect Locals / Watch — note unexpected values
  5. Step Over line-by-line until the value goes wrong; fix the logic
  6. Remove the breakpoint, re-run, and confirm the expected output

Sample Bug Pattern

total = 0
for i in range(1, 5):   # intended: sum 1..5
    total += i
print(total)  # prints 10, expected 15 — range stops early

Demo angle

Breakpoint inside the loop; watch i and total each step until learners spot the range mistake.

Test Your Knowledge!

Let's see what you've learned about the debugger

Q1: A breakpoint…

ADeletes your code
BPauses execution on that line
CCompiles Python to machine code
DFormats a hard drive
Answer: B — It pauses so you can inspect state.

Q2: Step Over is best when you…

AWant to enter every library call
BWant to run the current line without diving in
CWant to shut down the PC
DWant to ignore all errors forever
Answer: B — Step Over executes the line/call but stays at this level.

Q3: Compared with print-debugging, a debugger…

ACannot show variables
BLets you inspect live state without rewriting code
COnly works on websites
DReplaces version control
Answer: B — You pause and inspect without littering prints.
