Why Physical Computing?

The micro:bit is a tiny computer with real sensors and real outputs. Physical computing takes coding off the screen and into the physical world — press a button, shake it, and something actually happens in front of you.

Input → Action Logic

Shake detected

Pick random 1–6

Show number on LEDs

Every micro:bit program follows the same simple pattern: "when [input] happens, do [action]." A dice roll uses shake; a heat warning uses temperature compared against a threshold.

Flashing Code to the Device

In MakeCode / Python

  • Build the blocks (or Python) in the browser editor
  • Simulator shows a live preview
  • Click "Download" to save a .hex file

On the Device

  • Connect micro:bit via USB — it appears as a drive
  • Drag the .hex file onto that drive
  • Yellow LED flickers while flashing, then the code runs

Practical Task: Sensor-to-Action Build

  1. Choose your project: shake → dice roll or temperature → warning
  2. Open the MakeCode editor and start a new project
  3. Add an "on shake" (or "on start" with a forever loop reading temperature) block
  4. Dice: use "pick random 1 to 6" and "show number". Warning: use "if temperature > threshold" and "show icon"
  5. Add a sound effect to reinforce the output (v2 boards)
  6. Test in the on-screen simulator first
  7. Connect the micro:bit by USB and download the .hex file to it
  8. Wait for the flashing LED to finish, then disconnect and test on battery power
  9. Demo your finished project to the group and explain the input→action logic

APrint it and scan the paper
BDownload a .hex file and drag it onto the USB drive
CSpeak the code aloud near it
DIt updates automatically overnight
Answer: B — Flashing means copying the .hex file onto the micro:bit's USB drive.

AThe compass
BThe microphone
