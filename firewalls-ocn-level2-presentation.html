What is a Firewall?

Definition

A firewall is a security system—hardware, software, or both—that monitors and controls incoming and outgoing network traffic based on a set of security rules.

Think of it like this...

Imagine a security guard at a building entrance. They check who comes in and who goes out, allow trusted visitors, and block anyone who looks suspicious—exactly what a firewall does for your network!

Why Firewalls Matter

Without a firewall, your network is like a house with no locks on the doors—anyone on the internet could try to get in.

The Inspection Process

  • Every packet of data arrives at the firewall
  • The firewall checks it against security rules
  • Allow — trusted traffic passes through
  • Deny / Block — unwanted traffic is stopped

Hardware vs Software Firewalls

Hardware Firewall

  • Physical device between internet & network
  • Protects all devices on the network
  • Often built into routers
  • Higher performance for busy networks
  • Used in homes, offices & data centres
  • Examples: Cisco ASA, pfSense appliances

Software Firewall

  • Program installed on a computer or server
  • Protects that individual device
  • Built into Windows, macOS & Linux
  • Easy to configure for beginners
  • Good for laptops and remote workers
  • Examples: Windows Defender Firewall, iptables

Types of Firewalls

Packet Filtering

Basic & Fast

Checks source/destination IP, port, and protocol. Allows or blocks each packet individually.

Simple routers, basic ACLs

Stateful

Tracks Connections

Remembers active sessions. Knows if a packet is part of an allowed conversation.

Most modern business firewalls

Next-Gen (NGFW)

Smart & Deep

Inspects applications, blocks malware, and detects threats using advanced analysis.

Palo Alto, Fortinet, CrowdStrike

Firewall Rules Explained

Rules tell the firewall what to allow and what to block. They are usually checked from top to bottom.

Action Source Destination Port Why?
ALLOW Any Web Server 443 (HTTPS) Customers need the website
ALLOW Office PCs Internet 80 / 443 Staff browsing & email
DENY Any Database 3306 Block public DB access
DENY Any Any Any Default: block everything else

What Firewalls Protect Against

Unauthorised Access

Malware & Worms

Can limit flood attacks that try to overwhelm your systems

Controls what can leave the network, reducing data leaks

Risky Services

Firewall Best Practices

Do

  • Use a default deny all rule, then allow only what you need
  • Keep firmware and software updated
  • Review logs regularly for unusual activity
  • Combine hardware + software firewalls
  • Document every rule and why it exists

Don't

  • Leave unused ports open "just in case"
  • Use default admin passwords
  • Ignore firewall alerts and logs
  • Rely on a firewall alone—use layered security
  • Allow remote admin from the whole internet

Golden Rule: Least Privilege

Only open what is absolutely necessary. Everything else should be blocked by default.

Real-World Example

Small Business: Local Dental Practice

Before

  • Home router with default settings
  • Patient records on an open network share
  • No logging of network activity
  • Staff using weak remote desktop access

After

  • Business firewall with rule-based policies
  • Database locked behind deny rules
  • VPN required for remote access
  • Windows Firewall enabled on every PC

Result

Blocked 12,000+ malicious probes in the first month, met GDPR requirements, and protected sensitive patient data.
