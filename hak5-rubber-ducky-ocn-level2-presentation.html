USB Attack Awareness

Understanding Rubber Ducky-style threats

Learning Objectives

By the end of this session, you will be able to:

  • Explain what a USB keystroke-injection device is at a high level
  • Describe why these tools appear in authorised security testing
  • Recognise risks of unknown USB devices in the workplace
  • Apply defensive policies and user awareness controls
  • State legal and ethical boundaries for security tools

Ethics & Legal First

Devices like the Hak5 USB Rubber Ducky are commercial security-testing tools. Using them on systems without explicit authorisation can be illegal and can get you expelled, fired, or prosecuted.

Classroom rule

We study these tools to defend networks and train awareness—not to attack friends, school systems, or employers.

What Is a Rubber Ducky?

It looks like a normal USB flash drive, but the computer often sees it as a keyboard. Once plugged in, it can type pre-programmed keystrokes very quickly.

Defences That Work

USB control policies

Block unapproved devices via endpoint tools

Least privilege

Limit what standard user accounts can run

User awareness

Never plug in found or gifted unknown USBs

Screen locks

Locked sessions reduce unattended injection risk

Physical security

Reception, CCTV, visitor badges, clean desk

Patch & EDR

Detect unusual keyboard activity / scripts

Workplace Policy Example

  • Only company-issued storage devices allowed
  • Guest machines for vendor demos—not corporate laptops
  • Report lost/found USB sticks to IT Security
  • Disable AutoRun / AutoPlay features
  • Security testing only via approved change requests

Awareness Scenario

“Free USB” at reception

During an authorised phishing/USB awareness campaign, labelled test sticks are left in public areas (with management approval).

Learning outcome

Staff who plug them in get a training page, not malware—and the organisation measures how many people need more awareness support.
