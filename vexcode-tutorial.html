Start the tutorial

What it is

VEXcode is the official coding environment for VEX robots.

Built by VEX Robotics, VEXcode lets you write programs in Blocks, Python, or C++ and send them to classroom robots such as VEX GO, IQ, and V5. Drag colourful blocks when you are learning; switch to text when you are ready for more control.

VEX GO

Early STEM builds. Snap-together parts and a gentle Blocks introduction.

VEX IQ

Middle-school competition kits. Sensors, drivetrains, and event-driven code.

VEX V5

High-school competition robots. Powerful motors, vision, and text coding.

Getting started

Open VEXcode and talk to your brain.

You can run VEXcode on Windows, macOS, Chromebook, iPad, or in a supported browser, depending on your platform. Match the app edition to the robot you are using.

  1. Install or launch the right edition

    Download VEXcode GO, IQ, or V5 from the official VEX site, or open the web version if your school uses it. The toolbox and device list change to match that platform.

  2. Power the robot and connect

    Turn on the Brain. Connect with USB, or pair over Bluetooth / VEXlink when available. In VEXcode, open the Devices or Robot panel and confirm the Brain shows as connected.

  3. Configure motors and sensors

    Add each motor, drivetrain, bumper, distance sensor, or inertial sensor to the project configuration so Blocks and Text know which ports to use.

  4. Start a new project

    Choose Blocks for visual coding, or Python / C++ for text. Save early — competition day is a bad time to lose a drivetrain routine.

The workspace

Know the four places you live in VEXcode.

Once connected, almost every task happens in the same layout: toolbox, canvas, toolbar, and monitor.

when started
drive forward 200 mm
turn right 90°
wait 1 second
if bumper pressed?
print "Hit!"
  • Toolbox Categories of blocks or a text API reference — Motion, Sensing, Control, Events, and more.
  • Coding canvas Where you snap blocks together or type statements. Programs usually begin with a “when started” event.
  • Toolbar Download / Run, stop, help, examples, and project settings live here.
  • Monitor & console Watch sensor values and printed messages while the robot runs — essential for debugging.

Blocks or text

Same robot. Two ways to speak to it.

Start with Blocks to learn structure. Move to Python or C++ when you need loops, functions, and finer motor control.

Blocks

Snap puzzle pieces. Great for sequencing, conditionals, and classroom demos. Harder to mistype a command.

when started
drive forward

Python (example)

Readable text coding used widely in V5 and IQ classrooms once students leave Blocks.

# Drive forward, then turn
drivetrain.drive_for(
    FORWARD, 200, MM
)
drivetrain.turn_for(
    RIGHT, 90, DEGREES
)

First program

Make the robot drive a short square.

This is the classic “it works” project: move, turn, repeat. Clear space on a desk or floor before you run it.

01

Add a drivetrain in Devices

Configure left and right motors (and wheel size / track width if prompted). Without this, drive blocks have nowhere to send power.

02

Stack motion under “when started”

Drive forward a set distance, turn 90°, and repeat four times — or use a repeat loop around one side of the square.

03

Download to the Brain

Press Download (or Slot download on V5). Keep the cable connected until VEXcode reports success.

Run

Run from the Brain or the app

Start the program on the Brain screen, or use the VEXcode run control when tethered. Watch corners — slow the drive speed if it skids.

Tip

Print while you learn

Add a print or screen block after each turn. If the square fails, the last message on the console tells you where the stack stopped.

Sensors & events

Stop guessing — let the robot feel the world.

Once open-loop driving works, add sensing. Bumper switches, distance sensors, gyros / inertial sensors, and optical sensors turn a script into a reactive behaviour.

  1. Wait until a condition is true

    Example: drive forward until the distance sensor reports less than 100 mm, then stop. This is more reliable than a fixed “drive for 3 seconds.”

  2. Use events for buttons and bumps

    “When bumper pressed” or controller button events keep the main stack clean and make teleop (driver control) programs easier to read.

  3. Watch values live

    Open the monitor. Wave your hand in front of a distance sensor and confirm the number changes before you trust it in a competition routine.

Practice

Three challenges that teach the real skills.

Work through these in order. Each one forces a new VEXcode habit: sequencing, sensing, or structure.

Challenge A · Sequencing

Tape line out-and-back

Drive forward along a taped line for one metre, wait one second, reverse to the start. Tune distance and speed until the robot parks on the mark.

Challenge B · Sensing

Stop before the wall

Drive toward a wall or box and stop using a distance sensor — no timed drive. Print the measured distance when the robot halts.

Challenge C · Structure

Controller arcade drive

Map joystick axes to drivetrain velocity (Blocks or text). Add a bumper that stops all motors when pressed — a safety habit worth keeping.
