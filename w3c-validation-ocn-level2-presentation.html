W3C Validation

Validate HTML against web standards

OCN NI Level 2–3 — Website development

Learning Objectives

By the end of this session, you will be able to:

  • Explain what W3C validators check and why standards matter
  • Validate a demo page with the W3C Markup Validation Service
  • Interpret errors vs warnings in a validation report
  • Fix common markup issues (unclosed tags, wrong nesting, missing alt)
  • Decide when validation is good enough for classroom evidence

Core Ideas

Standards

  • HTML should follow agreed rules
  • Browsers are forgiving — bugs hide
  • Accessibility and SEO benefit
  • Employers expect clean markup

Typical finds

  • Unclosed or mismatched tags
  • Deprecated attributes
  • Images without alt text
  • Duplicate IDs / bad nesting

Demo Sequence

  1. Open the demo HTML page in VS Code
  2. Paste or upload into the W3C Markup Validator
  3. Read the first error carefully — fix causes, not symptoms
  4. Re-validate until errors are cleared (note warnings)
  5. Screenshot the clean report for evidence
