SQL Query Builder

Build SQL queries step by step — from SELECT to JOIN

OCN NI Level 2–3 — Databases

Learning Objectives

By the end of this session, you will be able to:

  • Read a simple schema (tables, keys, relationships)
  • Build SELECT queries one clause at a time
  • Filter with WHERE and sort with ORDER BY
  • Join two related tables to answer a business question
  • Explain what each part of a query does in plain English

Why Queries Matter

A database stores data; a query asks a precise question. SQL Query Builder demos show learners that SQL is a language of clear steps — not magic — and that each clause has a job.

Demo Sequence

  1. Show the projected schema (Customers, Orders) and sample rows
  2. Start with SELECT * from one table — then pick columns
  3. Add WHERE to filter (e.g. town = 'Belfast')
  4. Add ORDER BY and LIMIT / top-N
  5. Join Customers to Orders on CustomerID and answer: “Who ordered what?”

Build It Live

SELECT c.CustomerName, o.OrderDate, o.Total
FROM Customers AS c
JOIN Orders AS o ON c.CustomerID = o.CustomerID
WHERE c.Town = 'Belfast'
ORDER BY o.OrderDate DESC;

Tip

Project each line as you type it — ask the class what changes before running.

Test Your Knowledge!

Let's see what you've learned about SQL queries

Q1: The WHERE clause is used to…

AChoose which columns appear
BFilter which rows are returned
CSort the result set
DCreate a new table
Answer: B — WHERE filters rows; SELECT chooses columns; ORDER BY sorts.

Q2: A JOIN is needed when…

AYou only need one table
BRelated data lives in more than one table
CYou want to delete all rows
DSQL is offline
Answer: B — Joins connect tables using related key fields.

Q3: SELECT CustomerName FROM Customers; returns…

AEvery column from every table
BOnly the CustomerName column from Customers
CA sorted list of orders
DNothing without WHERE
Answer: B — You asked for one column from one table — that’s what you get.
