Native Vs Web Vs Hybrid

Compare three mobile app approaches

OCN NI Level 2–3 — Mobile apps

Learning Objectives

By the end of this session, you will be able to:

  • Define native, web, and hybrid mobile approaches
  • Compare performance, device features, and cost
  • Map examples to each approach
  • Recommend an approach for a simple classroom scenario
  • Spot trade-offs around stores, offline use, sensors, and updates

Core Ideas

Native

  • Built for one OS (Kotlin/Swift…)
  • Best performance and APIs
  • Separate codebases often
  • Store distribution common

Web & Hybrid

  • Web: browser / PWA using HTML/JS
  • Hybrid: web tech inside a shell
  • Faster cross-platform start
  • May limit deep device access

Demo Sequence

  1. Define the three approaches with a simple diagram
  2. Show or screenshot one example of each
  3. Fill a comparison grid: performance, APIs, cost, updates
  4. Give a scenario (school notice app) and choose an approach
  5. Challenge: what breaks if you pick wrong?
  6. Summarise trade-offs in one slide

Practical Notes

Choose native

Max performance / deep sensors

Choose web

Content sites, simple tools, PWAs

Choose hybrid

Cross-platform with some device APIs

Store rules

Fees, review, platform policies
