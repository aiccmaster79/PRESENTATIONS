Why Block Coding?

Scratch lets you snap together coloured blocks instead of typing syntax. It teaches the exact same computational thinking as "real" programming languages — sequence, selection and iteration — without syntax errors getting in the way.

How a Block Script Flows

Event fires

Check condition

Take action

Loop back

Almost every Scratch script follows this pattern: something triggers it, a condition is checked, an action happens, and a loop keeps it going until the game or quiz ends.

Game vs Quiz Project

2-Sprite Game

  • Player sprite moves with arrow keys
  • Target/enemy sprite moves or spawns
  • Score variable increases on "touching"
  • Forever loop keeps the game running

5-Question Quiz

  • "Ask and wait" block for each question
  • If/else checks the typed answer
  • Score variable +1 for each correct answer
  • "Say" block reveals the final score

Practical Task: Build Your Game or Quiz

  1. Choose: a 2-sprite game or a 5-question quiz
  2. Create/import your sprites and set the starting backdrop
  3. Create a score variable and set it to 0 on green flag
  4. Game: add movement with arrow keys and a "touching" check that changes score. Quiz: add five "ask and wait" questions with if/else scoring
  5. Add a loop (repeat or forever) so the game/quiz runs properly
  6. Add sound or a costume change as feedback for a correct action
  7. Show the final score with a "say" block
  8. Test the whole thing three times and note any bugs
  9. Fix each bug one at a time, retesting after every fix

CA variable

Q3: What is the best way to find bugs in your project?

AOnly test once at the very end
BTest often and fix one bug at a time
CDelete the project and start again every time

Answer: B — Frequent testing and isolating one change at a time makes bugs far easier to trace.
