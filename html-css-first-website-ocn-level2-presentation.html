How a Browser Renders a Page

Read HTML

Build the DOM

Apply CSS Styles

Paint the Page

The browser reads your HTML top to bottom, builds a structure called the DOM, applies any CSS rules, then paints pixels on screen. That's why tag order and correct nesting matter.

A Simple Page in Code

<!DOCTYPE html>
<html>
<head>
  <title>About Me</title>
  <style>
    body { font-family: sans-serif; background:#141428; color:#fff; }
    h1 { color: #F7931E; }
    a { color: #00BDA5; }
  </style>
</head>
<body>
  <h1>About Me</h1>
  <img src="me.jpg" alt="My photo" width="150">
  <p>Hi, I'm learning IT at Level 2!</p>
  <a href="https://example.com">My favourite site</a>
</body>
</html>

Try it

Save this as index.html and double-click it — no server or internet connection required.

Practical Task: Build Your "About Me" Page

  1. Create a new folder and a file named index.html
  2. Add the HTML skeleton: <!DOCTYPE html>, <html>, <head>, <body>
  3. Add a page <title> and an <h1> heading with your name
  4. Add two <p> paragraphs describing yourself and your interests
  5. Add an <img> tag with a photo or placeholder image and an alt description
  6. Add an <a href=""> link to a favourite website
  7. Add a <style> block: set a background colour, a font, and colour your heading
  8. Save the file and open it by double-clicking it in your file browser
  9. Check it renders correctly, then tweak one CSS value and refresh to see the change

Test Your Knowledge!

Let's see what you've learned about HTML & CSS

Q1: Which tag creates a clickable link?

A<img>
B<a href="">
C<p>
D<style>
Answer: B — The <a href=""> tag defines a hyperlink.

Q2: Where can CSS rules be written?

AOnly in a separate program
BInline, in a <style> tag, or a linked file
COnly inside <img> tags
DCSS cannot be saved
Answer: B — CSS can live inline on a tag, inside a <style> block, or in a separate .css file.

Q3: What does a browser build from your HTML before painting the page?

AA spreadsheet
BThe DOM (Document Object Model)
CA PDF file
DAn executable program
Answer: B — The DOM is the structured tree of elements the browser builds from your HTML.
