Website Hosting

Publish a static website

OCN NI Level 2–3 — Website development

Learning Objectives

By the end of this session, you will be able to:

  • Explain what hosting is and how a public URL reaches files
  • Choose a simple static hosting option for a classroom site
  • Publish HTML/CSS/JS files to a live host
  • Test the live site on desktop and phone
  • Troubleshoot common publish issues (wrong folder, cache, 404)

Core Ideas

What hosting does

  • Stores your site files on a server
  • Serves them over HTTP/HTTPS
  • Gives you a public address
  • May provide HTTPS certificates

Static vs dynamic

  • Static: HTML/CSS/JS files
  • Dynamic: server code + databases
  • Classroom demos start static
  • Cheaper and simpler to publish

Demo Sequence

  1. Confirm the site works locally (open index.html)
  2. Create a host project (Netlify, GitHub Pages, or school host)
  3. Upload or deploy the folder with index.html at the root
  4. Open the live URL and hard-refresh
  5. Test a second page and a phone viewport
  6. Fix 404s caused by paths, filenames, or case sensitivity

Practical Notes

Root matters

index.html must be findable

Relative paths

Broken links appear after publish

HTTPS

Modern hosts enable TLS easily

Cache

Old files can stick until refresh
