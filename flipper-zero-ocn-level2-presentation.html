Ethics & Legal First

Flipper Zero is a legally sold multi-tool, but many of its functions — cloning an access card, replaying a radio signal, injecting keystrokes — are only lawful with the owner's explicit permission. Radio transmission is also subject to national frequency regulations.

Classroom rule

We study Flipper Zero to understand how physical and RF attacks work so we can defend against them — testing only on your own hardware, never on college doors, cars, or other people's devices.

What is a Flipper Zero?

Definition

Flipper Zero is a portable, open-source hardware pentesting tool shaped like a games console. It bundles several radio and hardware interfaces into one pocket-sized device, controlled through a simple on-screen menu.

Think of it like this...

It's a Swiss Army knife for signals — instead of carrying separate RFID readers, IR remotes, and radio analysers, a security tester carries one device that does a bit of everything.

Hardware Modules

Radio

Sub-GHz

Captures and replays garage doors, barriers, some car fobs

RFID

125 kHz Low-Frequency

Reads/emulates older unencrypted access cards and fobs

NFC

13.56 MHz High-Frequency

Interacts with MIFARE-style cards, tags and transit passes

Infrared

Universal IR Remote

Learns and replays TV, AC and appliance remote codes

iButton

Contact Key Reader

Reads Dallas/iButton contact-based access keys

BadUSB

GPIO & Keystroke Injection

Acts like a Rubber Ducky when connected over USB; GPIO pins for hardware hacking

A Typical Authorised Access-Card Test

Written Permission

Read the Card

Report Weakness

  1. Scope & consent — client confirms which doors/cards may be tested
  2. Read the card — check whether it's an unencrypted 125 kHz type
  3. Attempt emulation — see if Flipper Zero can replay the credential (lab copy only)
  4. Report — recommend upgrading to encrypted, rolling-code credentials if vulnerable

Flipper Zero vs Traditional Tools

Flipper Zero

  • One pocket-sized, all-in-one device
  • Low cost compared to a full tool kit
  • Open-source firmware, active community
  • Easy menu-driven interface

Traditional Kit

  • Separate RFID reader, SDR, IR blaster, etc.
  • Higher combined cost and bulk
  • Often more precise / higher power for specialist jobs
  • Steeper learning curve per tool

Defensive Takeaways

Upgrade legacy cards

Move from unencrypted 125 kHz to encrypted MIFARE DESFire or similar

Use rolling codes

Gate/garage remotes should use rolling, not static, codes

RF shielding & distance

Keep key fobs away from windows and use signal-blocking pouches

Monitor access logs

Flag duplicate or unusual card reads at access points
