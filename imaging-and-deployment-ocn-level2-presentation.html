Imaging & Deployment

Why technicians clone and deploy instead of building every PC by hand

OCN NI Level 2–3 — IT Support / Service Delivery

Learning Objectives

By the end of this session, you will be able to:

  • Explain what disk imaging and deployment mean
  • Contrast manual setup vs image-based rollout
  • Describe a simple capture → store → deploy workflow
  • List benefits for schools and businesses
  • Identify risks (wrong image, licences, drivers)

Scale Beats Repetition

Building 50 PCs by hand wastes time and creates inconsistent results. Imaging captures a gold build and applies it repeatedly.

Manual vs Image Deploy

Manual configure

  • Install OS each time
  • Click through apps per PC
  • Easy to miss updates
  • OK for one-off repairs

Image / deploy

  • Standard apps & settings
  • Faster fleet rebuilds
  • Fewer configuration drifts
  • Needs planning & tooling

Simple Demo Workflow

  1. Build a reference (gold) device: OS, updates, apps, policies
  2. Generalise/sysprep as required by the tooling
  3. Capture an image to network share or USB/SSD
  4. Boot target device from deployment media / PXE
  5. Apply image; inject drivers if the hardware differs
  6. Boot, join domain/workgroup, test — then seal for users
