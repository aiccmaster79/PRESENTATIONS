What-If Modelling

Test alternatives in a spreadsheet to support decisions

OCN NI Level 2–3 — Data Modelling / Analytics

Learning Objectives

By the end of this session, you will be able to:

  • Separate inputs, calculations and outputs in a model
  • Change drivers (price, volume, cost) to see impact
  • Compare scenarios (best / base / worst)
  • Explain results to a non-technical decision maker
  • Avoid hard-coding values inside formulas

Decisions Need Options

What-if modelling lets you ask: “What if price rises 10%?” or “What if we sell 50 fewer units?” Good models keep inputs visible and editable so scenarios are safe to explore without rewriting formulas.

Model Structure

Inputs

Assumptions you can change

Calculations

Formulas only — no typed results

Outputs

Profit, total, break-even

Scenarios

Best / base / worst cases

Demo Sequence

  1. Open a simple profit model: Price, Units, Unit Cost → Revenue, Cost, Profit
  2. Highlight input cells in one colour; formulas in another
  3. Change Units — watch profit update live
  4. Build three scenario columns (or Scenario Manager / a toggle cell)
  5. Ask the class: which lever moves profit most — price or volume?

Test Your Knowledge!

Let's see what you've learned about what-if modelling

Q1: Inputs in a model should be…

AHidden inside long formulas only
BVisible cells you can change to test scenarios
CDeleted after each use
DAlways text, never numbers
Answer: B — Editable assumptions make what-if exploration safe and clear.

Q2: Hard-coding 1.2 inside a formula is weak because…

AExcel forbids numbers
BYou cannot see or change the assumption easily
CIt makes charts impossible
DIt enables macros
Answer: B — Put assumptions in input cells and reference them.

Q3: Best / base / worst scenarios help you…

AAvoid all maths
BCompare outcomes under different assumptions
CRemove the need for formulas
DEncrypt the workbook
Answer: B — Scenarios frame risk and opportunity for decisions.
