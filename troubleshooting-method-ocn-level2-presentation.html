Learning Objectives

By the end of this session, you will be able to:

  • Follow a logical fault-finding flowchart from symptom to fix

Why Method Matters

Guessing wastes time and can make things worse. A logical method finds the real fault faster, avoids unnecessary part-swapping, and builds trust with users.

The Fault-Finding Flowchart

1. Identify Symptom

2. Isolate Cause

3. Test Theory

4. Fix It

5. Verify

If the test disproves your theory, go back to step 2 and isolate a different possible cause — don't jump straight to swapping parts.

Common PC Faults & First Checks

Symptom

  • No display on monitor
  • No sound from speakers
  • No Wi-Fi / internet
  • Can't log in
  • PC running very slow

First checks

  • Cable, input source, monitor power
  • Mute status, volume, correct output device
  • Wi-Fi toggle/switch, router, airplane mode
  • Caps Lock, correct account, keyboard layout
  • Background apps, disk space, restart

Documenting & Escalating

Log the symptom

Write down exactly what the user reported

Record what you tried

Note each test and its result

Confirm the fix

Get the user to test it themselves too

Escalate if stuck

Pass on your notes so no one repeats your steps

Test Your Knowledge!

Let's see what you've learned about fault-finding

Q1: What is the first step of the fault-finding method?

AReplace the motherboard
BIdentify the symptom
CReinstall the OS
DCall it unfixable
Answer: B — You can't isolate a cause until you clearly understand the symptom being reported.

Q2: A user says their speakers are "broken" but audio is simply muted. This shows the importance of…

ABuying new speakers immediately
BChecking the simple causes first
CIgnoring the user's report
DReinstalling Windows
Answer: B — Most reported faults have a simple cause — always rule those out before assuming hardware failure.

Q3: After fixing an issue, what should you always do?

AWalk away immediately
BVerify the fix actually works
CDelete your notes
DAssume it's fine forever
Answer: B — Verifying the fix (ideally with the user testing it too) confirms the problem is really solved.
