File Handling

Programs that read and write files

OCN NI Level 2–3 — Programming

Learning Objectives

By the end of this session, you will be able to:

  • Explain why programs need to persist data in files
  • Open, read, and write a simple text file in Python
  • Use with open(...) so files close safely
  • Handle a missing-file error gracefully
  • Distinguish text vs binary use cases at a basic level

Why File Handling?

Variables vanish when a program ends. Files store data for later — logs, settings, CSV exports, user saves.

Read

Load existing data into the program

Write

Create or overwrite a file

Append

Add lines without wiping the file

Close

Release the file — use a with-block

Errors

Missing paths and permissions

Formats

Plain text, CSV, JSON…

Core Pattern

# Write
with open("notes.txt", "w", encoding="utf-8") as f:
    f.write("Hello, Level 2\n")

# Read
with open("notes.txt", "r", encoding="utf-8") as f:
    content = f.read()
    print(content)

Tip

with closes the file even if an error occurs — prefer it over bare open/close.

Demo Sequence

  1. Show a prepared script that writes three lines to a text file
  2. Run it, then open the file in VS Code / Notepad to prove it exists
  3. Add a second program (or mode) that reads and prints each line
  4. Rename/move the file and show FileNotFoundError — then catch it
  5. Discuss append mode ("a") vs write mode ("w") with a live demo
  6. Optional: read a simple CSV of names/scores and print a total

Common Pitfalls

"w" overwrites

Write mode wipes existing content — warn learners

Working directory

Relative paths depend on where you run the script

Permissions

School profiles may block some folders

Encoding

Use utf-8 for predictable text
