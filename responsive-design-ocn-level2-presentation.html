Responsive Design

Webpages that adapt to screen size

Learning Objectives

By the end of this session, you will be able to:

  • Explain why one fixed-width layout fails on phones
  • Use the viewport meta tag and fluid widths in simple CSS
  • Apply a basic media query to change layout at a breakpoint
  • Test layouts with DevTools device mode
  • Identify common responsive issues (overflow, tiny tap targets)

One Site, Many Screens

Responsive design makes layouts adapt to viewport width — phones, tablets, laptops — instead of building separate sites for each device.

Minimal Building Blocks

<meta name="viewport" content="width=device-width, initial-scale=1">

.container { max-width: 960px; width: 90%; margin: auto; }

@media (max-width: 600px) {
  nav { flex-direction: column; }
}

Demo Sequence

  1. Open a deliberately non-responsive demo page (fixed wide layout)
  2. Resize / device mode — show horizontal scroll and cramped content
  3. Add viewport meta + max-width container; reload and compare
  4. Introduce a media query that stacks nav or cards on small screens
  5. Test at phone and desktop widths in DevTools
  6. Checklist: no horizontal scroll, readable text, tappable buttons

Quick QA Checklist

No sideways scroll

At common phone widths

Text readable

Without pinching to zoom

Tap targets

Buttons not tiny or overlapping

Images scale

Don’t blow past the viewport

Q3: A common responsive failure is…

AHorizontal scrolling on phones
BToo many CPU cores
CMissing HDMI
