Learning Objectives

By the end of this session, you will be able to:

  • Explain the difference between cloud documents and local files
  • Use comments to give feedback without editing content
  • Use version history to view and restore earlier work
  • Set share permissions — viewer, commenter and editor
  • Collaborate respectfully on one shared document

Cloud Docs vs Local Files

Local File

  • Saved on one device
  • Shared by emailing copies back and forth
  • Easy to end up with confusing versions
  • Lost if the device is lost or breaks

Cloud Document

  • Stored online, accessible from any device
  • Everyone edits the same live copy
  • Changes save automatically
  • Backed up and protected from device loss

Practical Task

Edit One Shared Document, Together

As a whole class, work live on one shared doc or sheet — explore comments, version history and permissions.

  1. Tutor creates one shared document and shares the link with the class
  2. Each learner adds a section under their own name
  3. Leave a comment on someone else's section instead of editing it
  4. Reply to a comment left on your own section
  5. Open version history to see the changes made so far
  6. Restore an earlier version, then redo the change on purpose
  7. One learner sets sharing to "view only" and tests it with another device

Test Your Knowledge!

Let's see what you've learned about cloud collaboration

Q1: What's a key benefit of cloud documents over emailing files back and forth?

AIt creates more confusing versions
BEveryone edits the same live document, avoiding version conflicts
CIt's always slower
DNo one can see any changes
Answer: B — There's one live copy, so there's no "which version is newest" confusion.

Q2: Version history lets you…

ADelete the file permanently
BSee and restore earlier versions of a document
CChange the file name only
DNothing useful
Answer: B — It's a safety net if something is deleted or changed by mistake.

Q3: Giving someone "Viewer" access means they can…

AEdit and delete content freely
BOnly view / read the document
CChange who else can access it
DNothing — they can't open it at all
Answer: B — Viewer access is read-only — safe for sharing without risking edits.
