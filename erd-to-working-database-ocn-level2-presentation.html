ERD to Working Database

Turn a simple ERD into real tables

OCN NI Level 2–3 — Databases

Learning Objectives

By the end of this session, you will be able to:

  • Read a simple entity-relationship diagram (ERD)
  • Translate entities and attributes into table fields
  • Choose primary keys and basic relationships
  • Build the tables in Access, MySQL, or SQLite
  • Insert sample data that matches the design

Design Before You Type

An ERD shows entities (things), attributes (details), and relationships. Building tables from a clear diagram prevents messy, duplicated data later.

Rule of thumb

One entity → one table; unique rows need a primary key.

Mini ERD Example

Student

StudentID, Name, Email

Course

CourseID, Title, Level

Enrolment

StudentID + CourseID

Keys

PK uniquely identifies rows

Relationship: a student can enrol on many courses; a course has many students — modelled with an enrolment table or link.

Demo Sequence

  1. Project the sample ERD and name entities, attributes, and relationships
  2. Create matching tables in Access/MySQL/SQLite
  3. Set primary keys (and foreign keys if the tool supports them)
  4. Enter a few sample rows that obey the relationships
  5. Run a simple query: list students on a given course
  6. Learners extend the ERD with one new attribute or entity and update the DB

Design Smells

Healthy design

  • Clear primary keys
  • Atomic fields (one fact each)
  • Relationships documented
  • Sample data validates rules

Trouble ahead

  • Everything in one giant table
  • Repeated student addresses everywhere
  • No unique IDs
  • Columns named “Data1” “Data2”
