Macro Recording

Record a simple macro to automate a repeated task

OCN NI Level 2–3 — Data modelling / analytics

Learning Objectives

By the end of this session, you will be able to:

  • Explain what a macro is in plain language
  • Enable the Developer tab / macro settings appropriately
  • Record a short macro for a repeatable format or cleanup
  • Run the macro and assign it to a button (optional)
  • Discuss risks: trusted sources only, .xlsm awareness

Teach Excel Your Shortcut

A macro records your keystrokes and clicks so Excel can replay them. For Level 2, recording (not writing VBA from scratch) is enough to automate tidy-up steps — bold headers, autosize columns, or filter a standard report view.

Demo Sequence

  1. Enable Developer tab; set macro settings to disable with notification
  2. Start Record Macro — name it e.g. FormatReport; store in This Workbook
  3. Perform 4–5 clear actions: header bold, filter on, autosize, freeze top row
  4. Stop recording
  5. Undo / reset the sheet — then run the macro to replay
  6. Optional: insert a button and assign the macro; save as .xlsm

Macro Recording

Automate a repeated task in a spreadsheet

OCN NI Level 2–3 — Data modelling / analytics

Learning Objectives

By the end of this session, you will be able to:

  • Explain what a macro is and when it saves time
  • Record a simple Excel macro for a repeated formatting or copy task
  • Run the macro and confirm it behaves as expected
  • Identify risks of macros (security, fragile steps) and safe practice
  • Decide when a formula or Power Query is better than a macro

What Is a Macro?

A macro records a sequence of actions (clicks, typing, formatting) so you can replay them with one shortcut or button — useful for repetitive spreadsheet work.

Tip

Start with a short, reliable task (e.g. format a header row). Long, branching macros get fragile fast.

Why Automate?

Speed

Consistency

Same steps every time — fewer mistakes

Caution

VBA behind scenes

Recording writes code you can inspect later

Not always best

Formulas often beat macros for calculations

Demo Sequence

  1. Open the prepared sample workbook (sales or student marks model)
  2. Enable the Developer tab if needed (Excel Options → Customize Ribbon)
  3. Start Record Macro — name it clearly (e.g. FormatHeader)
  4. Perform the repeated task once (format row, copy values, add borders)
  5. Stop recording, then run the macro from the Macros dialog or a button
  6. Discuss: what would break if columns moved? When to use formulas instead?
