What is Virtualisation?

Definition

Virtualisation is the process of creating a virtual (software-based) version of something—such as a computer, operating system, storage device, or network—rather than using physical hardware.

Think of it like this...

Imagine having one powerful computer that can act like multiple separate computers, each running its own operating system and applications, all at the same time!

Physical vs Virtual

Physical Computing

  • One operating system per physical machine
  • Hardware tied to specific tasks
  • High cost for additional servers
  • Limited flexibility and scalability
  • Hardware failures cause downtime
  • Resources often underutilised

Virtual Computing

  • Multiple OS on one physical machine
  • Flexible resource allocation
  • Lower hardware costs
  • Easy to scale up or down
  • Quick disaster recovery
  • Maximum resource utilisation

Key Component: Hypervisor

The hypervisor (also called Virtual Machine Monitor) is the software that:

  • Creates and manages virtual machines
  • Allocates physical resources (CPU, RAM, storage)
  • Isolates VMs from each other
  • Enables multiple OS to share hardware

Types of Hypervisors

Type 1: Bare-Metal

Runs directly on hardware without a host OS

  • Higher performance & efficiency
  • Better security (smaller attack surface)
  • Used in enterprise data centres
  • Examples: VMware ESXi, Hyper-V Server, Xen
Enterprise & Cloud Use

Type 2: Hosted

Runs on top of an existing OS

  • Easier to set up and use
  • Good for testing & development
  • Runs alongside other applications
  • Examples: VirtualBox, VMware Workstation

Cloud computing is built on virtualisation technology. Major cloud platforms like AWS, Azure, and Google Cloud use hypervisors to provide virtual machines as a service.

Cloud Service Models

IaaS

Infrastructure as a Service

Rent virtual servers, storage & networking. You manage OS and apps.

AWS EC2, Azure VMs, Google Compute

Business Benefits: Scalability

Scale Up

Instantly add more CPU, RAM, or storage to handle increased demand—no hardware purchase needed!

Example: Online shop scales up servers for Black Friday sales.

Scale Down

Reduce resources when demand drops. Only pay for what you use!

Example: Reduce overnight capacity when fewer users are active.

Key Benefit: Pay-As-You-Go

Cloud services typically use a pay-as-you-go pricing model, allowing businesses to pay only for the resources they actually use—reducing infrastructure costs significantly.

A VPN creates a secure, encrypted tunnel between your device and a remote server, protecting your data as it travels across the internet.
