Learning Objectives

By the end of this session, you will be able to:

  • Build a simple trace table with columns for variables and output
  • Dry-run a short program on paper before executing it
  • Predict variable values after each statement
  • Compare predictions with the debugger or a live run
  • Spot logic errors that appear as unexpected table rows

What Is a Trace Table?

A trace table is a paper (or spreadsheet) dry-run of code: each row is a step; columns hold variables and any output. It trains you to think like the computer.

Tip

Use one column per variable. Update only what changes on that line — leave others blank or carry forward clearly.

Example Trace

n = 3
total = 0
while n > 0:
    total = total + n
    n = n - 1
print(total)

Demo Sequence

  1. Show a short Python snippet (loop + accumulator) on screen
  2. Draw blank columns: Line / var1 / var2 / Output
  3. Learners fill the table in pairs without running the code
  4. Reveal the completed table row by row; discuss disagreements
  5. Run the code (or step in the debugger) to confirm
  6. Change one line (e.g. start value) and re-trace — emphasise prediction skill
