Phishing Anatomy

Dissecting a fake phishing email safely

OCN NI Level 5 — Cyber Security Demo

Learning Objectives

By the end of this session, you will be able to:

  • Identify common parts of a phishing message
  • Spot red flags in sender, links and urgency
  • Describe safe reporting actions for users
  • Explain why MFA and password managers reduce impact
  • Avoid interacting with real suspicious mail in class

Ethics & Scope

We only dissect prepared safe examples. Do not open real suspicious attachments in class. Report suspected phishing via your organisation’s official channel.

Authorised use only

Only use tools on systems and networks you are authorised to test.

What Attackers Want

Phishing tricks people into revealing credentials, installing malware or paying fraudulently. The email is a social engineering wrapper around a technical goal.

Tip

Slow down — urgency is often the weapon.

Anatomy Checklist

From address

Lookalike domains

Reply-to mismatch

Different from display name

Hover links

URL ≠ friendly text

Urgency / threat

Account locked! Pay now!

Unexpected attachments

Macros / zips

Tone & errors

Odd branding/language

Demo Sequence

  1. Open a prepared safe phishing example on the projector
  2. Highlight display name vs real address
  3. Hover (don’t click) to reveal the true link destination
  4. Call out urgency language and requests for secrets
  5. Show the correct report button / forward-to-security process
  6. Contrast with a legitimate message from the same brand (if available)

Test Your Knowledge!

Let's see what you've learned about phishing awareness

Q1: A common phishing red flag is…

ACalm, expected messages from known contacts via official channels
BUrgent threats plus a lookalike sender domain
CMeeting invites you already expected
DPassword manager notifications you triggered
Answer: B — Urgency + impersonation is a classic pairing.

Q2: If unsure about a link you should…

AClick quickly before it expires
BNavigate via a known bookmark / official app and report the mail
CForward it to all classmates to test
DDisable MFA so login is easier
Answer: B — Out-of-band verification beats email links.

Q3: MFA reduces phishing impact because…

AAttackers never try again
BA stolen password alone is often not enough
CIt deletes all emails
DIt replaces antivirus completely
Answer: B — Second factors block many credential-only attacks (though MFA phishing exists — stay vigilant).
