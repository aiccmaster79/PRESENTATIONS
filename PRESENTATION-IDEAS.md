# Presentation Ideas — OCN NI Level 2 Diploma in Information Technology

A backlog of **practical, classroom-ready presentation topics** for Level 2 IT
learners. Each idea is chosen so it can be *shown and done*, not just talked
about — every one pairs a short teaching deck with a hands-on task, a live demo,
or something learners can build in a lesson.

**How to read this list**
- **Practical hook** — the hands-on thing learners actually do.
- **Kit needed** — what you need to run it (kept cheap / already in most IT rooms).
- **Assessment fit** — how it supports evidence for the OCN NI Level 2 units.

Ideas are grouped to match the existing library: **Core IT**, **Cyber
Security**, **Emerging Technology**, plus two new practical strands —
**Digital Skills & Employability** and **Project / Maker** — that round the
course out.

## Core IT (hardware, software, systems)

### 1. Inside the PC — Identify & Handle Components
- **Practical hook:** Pass round real parts (RAM stick, HDD vs SSD, CPU, GPU, PSU); learners label a blank diagram and sort parts into "input / output / storage / processing".
- **Kit needed:** A retired desktop or a box of spare parts, anti-static wrist strap.
- **Assessment fit:** Extends *Building Computers* — component ID and safe handling (ESD).

### 2. Storage Showdown — HDD vs SSD vs Cloud vs USB
- **Practical hook:** Time a file copy to a USB stick vs an SSD; compare cost-per-GB in a shared spreadsheet; decide the right storage for three scenarios (photographer, school, gamer).
- **Kit needed:** USB sticks, a PC, a spreadsheet.
- **Assessment fit:** Storage types, capacity, and selecting fit-for-purpose hardware.

### 3. Operating Systems Face-Off — Windows vs macOS vs Linux vs ChromeOS
- **Practical hook:** Boot a Linux live USB and complete 5 tasks (open a browser, create a folder, find settings); compare against Windows.
- **Kit needed:** One live-boot USB (Ubuntu/Mint), a spare PC.
- **Assessment fit:** OS purpose, features, and choosing an OS for a use case.

### 4. Safe Setup — Ergonomics, Health & Safety at the Workstation
- **Practical hook:** Learners audit their own desk against a checklist (screen height, seating, cable safety, lighting) and photograph one fix they make.
- **Kit needed:** The classroom itself, phones/cameras.
- **Assessment fit:** Health & safety, responsible use of IT equipment.

### 5. Troubleshooting Like a Technician — The Fault-Finding Method
- **Practical hook:** Instructor breaks 5 things (unplugged monitor, muted audio, disabled Wi-Fi, wrong input, caps-lock login); teams race to diagnose using a logical flowchart.
- **Kit needed:** A few PCs you can safely mis-configure.
- **Assessment fit:** Diagnosing and resolving common IT problems; support skills.

### 6. Backups & Data Recovery — The 3-2-1 Rule
- **Practical hook:** Learners back up a folder to two locations, then "accidentally" delete it and restore it; discuss what a real ransomware victim would lose.
- **Kit needed:** USB sticks / cloud folder.
- **Assessment fit:** Data protection, responsible use, links to cyber unit.

## Cyber Security

### 7. Password Power — Build & Test a Strong Password
- **Practical hook:** Use an offline "how-long-to-crack" estimator; learners strengthen a weak password step by step, then set up a passphrase and enable 2FA on a demo account.
- **Kit needed:** Browser (offline strength checker), demo email account.
- **Assessment fit:** Authentication, account security, safe online behaviour.

### 8. Spot the Phish — Email & Message Scam Clinic
- **Practical hook:** Learners score 10 real-style emails/texts as "safe or scam", circling the red flags (sender, links, urgency, spelling); build a class "spot-the-scam" checklist.
- **Kit needed:** Printed or on-screen sample messages.
- **Assessment fit:** Social engineering awareness, safe use of IT — high employability value.

### 9. Social Media & Digital Footprint — What Can Strangers Find?
- **Practical hook:** Learners google themselves / a made-up persona and map what's public; then lock down privacy settings on one account live.
- **Kit needed:** Phones/PCs.
- **Assessment fit:** Responsible/ethical use of IT, online safety, digital citizenship.

### 10. Malware Museum — Viruses, Ransomware, Worms & Trojans
- **Practical hook:** Card-sort activity matching malware type → how it spreads → how to defend; watch a *safe* sandboxed ransomware demo (video) and list the defences that would have stopped it.
- **Kit needed:** Prepared cards, projector.
- **Assessment fit:** Threat types and countermeasures; pairs with *Firewalls* and *Metasploit* decks.

### 11. Wi-Fi Security — Securing a Home / Small-Office Router
- **Practical hook:** Walk through a router admin page (screenshots or a real spare router): change default password, set WPA3, hide/rename SSID, set up a guest network.
- **Kit needed:** A spare router or admin-panel screenshots.
- **Assessment fit:** Network security in practice; complements *Building Networks* and *Firewalls*.

### 12. Encryption Made Simple — Locking Data with a Cipher
- **Practical hook:** Learners encrypt/decrypt messages by hand with a Caesar cipher, then see the padlock/HTTPS in a browser and explain what it protects.
- **Kit needed:** Paper, a browser.
- **Assessment fit:** Confidentiality, why encryption matters — links to blockchain & quantum decks.

### 13. Staying Legal — Data Protection, Copyright & Acceptable Use
- **Practical hook:** Teams judge 6 short scenarios ("Is this allowed?") against GDPR / copyright / the college AUP and justify a verdict.
- **Kit needed:** Scenario cards.
- **Assessment fit:** Legal and ethical responsibilities in IT.

## Emerging Technology (A/651/0447)

### 14. AI in Your Pocket — Prompting a Chatbot Effectively
- **Practical hook:** Learners write a weak prompt then improve it 3 times; compare outputs; list 3 tasks AI is good at and 3 it gets wrong (spot a "hallucination").
- **Kit needed:** Access to a chatbot or screen-recorded examples.
- **Assessment fit:** Extends *AI & Machine Learning* — benefits, risks, responsible use.

### 15. Smart Home Build — Designing an IoT Room
- **Practical hook:** Teams design a smart classroom/home on paper: pick 5 sensors/devices, draw the network, list one security risk per device.
- **Kit needed:** Paper/whiteboard (or a micro:bit for a live temperature sensor demo).
- **Assessment fit:** Extends *Internet of Things* — sensors, value, and security.

### 16. Deepfakes & Misinformation — Can You Trust What You See?
- **Practical hook:** Learners compare real vs AI-generated images/audio and vote; build a checklist for verifying online content.
- **Kit needed:** Curated example media.
- **Assessment fit:** Impact of emerging tech on society; digital literacy and ethics.

### 17. Wearables & Health Tech — Data on Your Wrist
- **Practical hook:** Look at what a smartwatch tracks; learners map the data flow (sensor → phone → cloud) and debate who should own health data.
- **Kit needed:** A smartwatch/fitness band or spec sheet.
- **Assessment fit:** Emerging tech adoption factors, benefits vs privacy risks.

### 18. Robots & Automation — Which Jobs Change?
- **Practical hook:** Teams take a familiar job (checkout, warehouse, driver) and design how automation changes it; predict the *new* jobs it creates.
- **Kit needed:** None (discussion + poster).
- **Assessment fit:** Business/society impact of emerging technology.

### 19. Green IT — Sustainable Technology & E-Waste
- **Practical hook:** Learners audit the room's devices for power use, calculate a rough yearly cost, and plan an e-waste recycling route for old kit.
- **Kit needed:** Power ratings, a spreadsheet.
- **Assessment fit:** Responsible use of IT, society/environment impact — strong cross-topic link.

## Digital Skills & Employability (new strand)

### 20. Build Your First CV in a Word Processor
- **Practical hook:** Learners create a formatted, one-page CV (headings, bullet points, consistent styling) and export to PDF.
- **Kit needed:** Word processor.
- **Assessment fit:** Productivity software skills, employability — directly usable evidence.

### 21. Spreadsheets That Work — Budgets, Formulas & Charts
- **Practical hook:** Build a personal/event budget: enter data, use `SUM`/`AVERAGE`, add a chart, apply conditional formatting to flag overspend.
- **Kit needed:** Spreadsheet software.
- **Assessment fit:** Data handling and productivity software — core Level 2 evidence.

### 22. Design a Poster / Flyer — Visual Communication
- **Practical hook:** Learners produce an event poster following a simple design brief (audience, message, brand colours) and give peer feedback.
- **Kit needed:** Canva/Publisher/Slides or similar.
- **Assessment fit:** Digital media creation, communicating for an audience.

### 23. Email & Online Etiquette for the Workplace
- **Practical hook:** Rewrite 3 "bad" emails to be professional; practise CC/BCC, subject lines, and attaching files safely.
- **Kit needed:** Email client (real or mock-up).
- **Assessment fit:** Digital communication, employability, safe use.

### 24. Cloud Collaboration — Work on One Document Together
- **Practical hook:** A whole team edits one shared doc/sheet live; explore comments, version history, and share permissions (view vs edit).
- **Kit needed:** Google Workspace / Microsoft 365.
- **Assessment fit:** Cloud tools, collaboration, responsible sharing — links to *Virtualisation* deck.

### 25. IT Careers & Pathways in Northern Ireland
- **Practical hook:** Learners map 5 IT roles (support, networking, cyber, dev, data) to skills and local employers/apprenticeships; pick one and write a mini action plan.
- **Kit needed:** Careers websites / local employer list.
- **Assessment fit:** Progression, employability, personal development planning.

## Project / Maker (build-something lessons)

### 26. Build a One-Page Website with HTML & CSS
- **Practical hook:** Learners hand-code a simple "About Me" page (headings, image, link, a splash of CSS colour) and open it in a browser.
- **Kit needed:** A text editor + browser (no internet required).
- **Assessment fit:** Introductory coding, digital creation, problem-solving.

### 27. Block Coding a Game or Quiz (Scratch)
- **Practical hook:** Build a working 2-sprite game or a 5-question quiz with score-keeping; test and debug it.
- **Kit needed:** Scratch (offline editor works).
- **Assessment fit:** Computational thinking, logic, testing and iteration.

### 28. micro:bit Mini-Project — Sensor to Action
- **Practical hook:** Program a micro:bit to react to input (shake → dice roll, temperature → warning); flash it and demo it.
- **Kit needed:** BBC micro:bits (common in NI schools/colleges).
- **Assessment fit:** Physical computing, IoT concepts made real, testing.

### 29. Plan & Present a Small-Office Network
- **Practical hook:** Teams design a network for a 6-person business on paper/diagram tool (devices, IP plan, security), then pitch it in 3 minutes.
- **Kit needed:** draw.io / paper.
- **Assessment fit:** Applies *Building Networks* + *Firewalls* into a design task with a presentation deliverable.

### 30. Cable to Cloud — End-to-End "How the Internet Works" Build
- **Practical hook:** A relay activity: crimp a cable → plug into a switch → get an IP → load a page → trace the request. Each station is a mini-station learners rotate through.
- **Kit needed:** Cables/crimp tool (already in library), a switch, PCs.
- **Assessment fit:** Ties together *Making 568B Cables*, *Building Networks*, and internet fundamentals.

## Quick pick — highest practical value first

If you only build a handful next, these give the most hands-on payoff per lesson:

1. **Spot the Phish (#8)** — huge real-world value, zero kit.
2. **Password Power (#7)** — everyone uses it that day.
3. **Troubleshooting Like a Technician (#5)** — pure practical skills.
4. **Build Your First CV (#20)** — immediate employability evidence.
5. **Build a One-Page Website (#26)** — visible "I made this" result.
6. **micro:bit Mini-Project (#28)** — best low-cost maker buzz.
