Virtual Networks

NAT, bridged and host-only networking for VMs

OCN NI Level 2–3 — Virtualisation

Learning Objectives

By the end of this session, you will be able to:

  • Compare NAT, bridged and host-only modes
  • Choose a mode for lab safety vs LAN access
  • Demonstrate connectivity differences with ping tests
  • Explain how NAT hides VMs behind the host
  • Design a simple isolated practice network

Why Virtual NICs Matter

Every VM needs a network mode. The wrong choice can expose a lab VM to the real LAN — or block the internet downloads you need for class.

Tip

Default to isolated modes when practising risky tools.

Three Common Modes

ModeVM can reachTypical use
NATInternet via host; LAN hosts usually can’t initiate to VMEasy internet for updates
BridgedAppears as another device on the real LANLAN services / demos needing real IPs
Host-onlyHost + other host-only VMs (no LAN/internet by default)Isolated labs
Internal (some products)VMs on that network onlyUltra-isolated sandboxes

Tip

Product names vary slightly (VMware / VirtualBox / Hyper-V) — concepts are the same.

NAT vs Bridged

NAT

  • Host shares its connection
  • VM gets private translated address
  • Simple outbound access
  • Inbound from LAN often blocked
  • Safer default for many labs

Bridged

  • VM joins the physical network
  • Gets LAN DHCP address typically
  • Visible to other LAN devices
  • Needed for some server demos
  • Follow classroom network policy

Demo Sequence

  1. Start with host-only: show VM ↔ host ping, no internet
  2. Switch to NAT: show internet works; note VM IP range
  3. Switch to bridged (if policy allows): show LAN IP via DHCP
  4. From another LAN PC, compare reachability in each mode
  5. Return VM to host-only/NAT for safe classroom use
  6. Sketch when each mode is appropriate

Test Your Knowledge!

Let's see what you've learned about virtual networking modes

Q1: Host-only networking is mainly for…

AMaking the VM a public web server on the internet
BIsolating VMs with the host for safe labs
CReplacing the physical switch permanently
DDisabling all NICs forever
Answer: B — Host-only keeps traffic off the production LAN by default.

Q2: NAT typically lets a VM…

AAppear with a real LAN IP and accept all inbound LAN connections easily
BReach the internet through the host while staying behind translation
CIgnore TCP/IP entirely
DCrimp cables automatically
Answer: B — NAT shares the host’s outbound path with address translation.

Q3: Bridged mode should be used carefully because…

AIt joins the VM to the real network like another PC
BIt deletes snapshots
CIt turns off the monitor
DIt only works with printers
Answer: A — The VM becomes a LAN peer — follow policy and isolate risky workloads.
